# 皇室战争锦标赛排名计算器

> 打多少胜才能稳进前 900？这个工具帮你算。

一个基于数学模型的锦标赛排名预测工具，帮助皇室战争（Clash Royale）玩家在启程锦标赛、入围锦标赛等赛事中提前规划所需胜场。

## 在线体验

- **Vercel**：https://crccjinbiaosai.vercel.app
- **GitHub Pages**：https://treerobin06.github.io/cr-tournament-calc/

## 功能一览

### 四种查询模式

| 模式 | 输入 | 输出 |
|------|------|------|
| **胜场 → 排名** | 参赛人数 + 我的胜场 | 预估排名（乐观/保守）+ 晋级概率 |
| **名次 → 胜场** | 参赛人数 + 目标名次 | 每个胜场的晋级概率，标注 95% 安全胜场 |
| **鲁棒性分析** | 胜场 + 人数范围 | 不同参赛规模下是否安全晋级 |
| **安全人数** | 胜场 + 目标名次 | 最多多少人参赛你仍能安全晋级 |

### 可视化图表

- **胜场分布曲线** — 各胜场的玩家人数分布（钟形曲线）
- **累计排名曲线** — 胜场与排名的对应关系
- **衰减比例图** — 每多赢一场，剩余人数缩减多少
- **鲁棒性热力图** — 不同参赛人数 × 不同胜场的晋级概率

### 蒙特卡罗模拟

支持 Bradley-Terry 禀赋模型，模拟有实力差异时的真实对战：

- 可调 κ 参数控制实力差异程度
- 支持多次模拟取平均（1~50 次）
- 展示冠军胜场、目标排名胜场的统计数据

### 其他功能

- **预测最终排名** — 输入当前战绩（如 15-0），预测最终排名区间
- **数学洞察** — 展示背后的负二项分布公式和衰减规律
- **完整分布表** — 每个胜场的人数、累计人数、衰减比

## 数学原理

每位玩家的胜场数服从 **负二项分布 NegBin(r, 0.5)**：

```
P(W=k) = C(k+r-1, r-1) × (0.5)^(k+r)
```

其中 `r` 是命数（通常为 5），`k` 是胜场数。

递推公式（高效计算）：

```
P(W=0) = (0.5)^r
P(W=k+1) = P(W=k) × (k+r) / (2×(k+1))
```

晋级概率使用二项分布的正态近似：

```
X_k ~ Binomial(N, S(k))
P(晋级) = P(X_k ≤ target - 1)
```

详细推导见 [设计文档](docs/superpowers/specs/2026-04-05-cr-tournament-calculator-design.md)。

## 技术栈

- **React 18 + TypeScript** — 前端框架
- **Vite** — 构建工具
- **Tailwind CSS + shadcn/ui** — 样式和组件库
- **Recharts** — 图表渲染
- **KaTeX** — 数学公式渲染
- **Web Worker** — 蒙特卡罗模拟（不阻塞 UI）

所有计算在浏览器本地运行，不需要后端服务器，不收集任何用户数据。

## 模型局限性

- **同质胜率假设**：理论计算假设所有人胜率为 50%。真实对战中强者胜率更高，但对前 900 名门槛影响较小
- **退赛行为简化**：部分玩家可能提前放弃，当前用"打满比例"参数近似
- **截止时间忽略**：假设大部分人打到出局。实际比赛有时间限制

建议将计算结果视为 **参考区间**，在边界胜场 ±1 范围内做决策缓冲。
