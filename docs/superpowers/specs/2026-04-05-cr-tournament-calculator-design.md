# 皇室战争锦标赛排名计算器 — 设计文档

## 概述

一个纯前端 Web 应用（产品工具，非科研论文），用于**估算**皇室战争（Clash Royale）锦标赛中"打到多少胜大概率能进前 N 名"。基于负二项分布的数学模型作为零阶基线，所有输出附带不确定性区间。秒级响应，同时提供可选的蒙特卡罗模拟验证。

**重要 disclaimer**：本工具基于"所有玩家胜率 50%"的同质化假设，是理论基线估算而非精确预测。实际排名会受玩家实力差异、退赛行为、截止时间等因素影响。输出结果应视为"参考区间"而非"确定阈值"。

## 核心场景

用户（CR 竞技玩家）需要在比赛前快速获得参考估算：
- "这次比赛大概 24 万人参加，进前 900 名大概需要多少胜？（多大概率够？）"
- "我不确定有多少人参赛，15 胜在什么人数范围内大概率安全？"
- "我现在 15-0，继续打下去最终大概排哪里？"（附带 disclaimer：此预测假设后续胜率为 50%，实际强者表现会更好）

## 游戏规则建模

### 锦标赛规则
- 每人有 r 条命（默认 5），即最多 r 负出局
- 配对机制：只有胜场差 ≤ 1 的玩家才能匹配（此规则在 50% 胜率下对分布无影响，已通过大量模拟验证）
- 排名规则：胜场多排前面；同胜场时，负场少排前面
- 有截止时间：绝大部分人打到出局，极少数顶尖高手可能截止时仍在打

### 数学模型

**单人胜场分布 — 负二项分布 NegBin(r, 0.5)**

```
P(W=k | r命) = C(k+r-1, r-1) × (0.5)^(k+r)
```

**尾分布（≥ k 胜的概率）**

```
S(k) = Σ P(W=j), j=k..∞
截断条件：剩余尾质量 < 0.1/N（与业务目标对齐，而非固定阈值）
```

**数值计算：递推公式（避免直接组合数的溢出问题）**

```
P(W=0) = (0.5)^r
P(W=k+1) = P(W=k) × (k+r) / (2×(k+1))
```

**混合命数（处理提前放弃）**

```
P_mixed(W=k) = α × P(W=k | r=5) + (1-α) × P(W=k | r=4)
α = 打满比例（可调，默认 0.9 = 90% 打满 5 命）
```

当 r 和 提前放弃的命数 不是 4 和 5 时，公式泛化为：
```
P_mixed(W=k) = α × P(W=k | r=r_full) + (1-α) × P(W=k | r=r_quit)
r_quit = r_full - 1（提前放弃者少打一命）
```

**注意**：α 参数是敏感性分析工具，不是精确校正项。真实退赛行为可能与战绩相关（弱者更易放弃），但在缺乏真实数据的情况下，二点混合作为一阶近似足够用于参考。

### 排名计算

**给定战绩 (W胜, L负) 的排名：**

```
rank = (> W 胜的人数) + (W 胜但 < L 负的人数) + 1
     = N × S_mixed(W+1) + Σ_{j=0}^{L-1} [N × P_record(W, j)] + 1
```

其中 P_record(W, j) = 在 j 负出局的玩家中恰好 W 胜的概率。

**简化情况（打满 r 负出局的玩家）：**

由于绝大部分人以 r 负出局，同胜场的人几乎都是 r 负，tiebreaker 主要影响极头部。对于前 900 名门槛的估算：
```
保守排名（最坏情况，r 负）: rank ≈ N × S_mixed(k)
乐观排名（0 负）: rank ≈ N × S_mixed(k+1) + 1
```

**概率化输出（解决假确定性问题）：**

边界附近 k 和 k+1 胜的差距可能很大。应输出晋级概率而非裸阈值：
```
令 X_k ~ Binomial(N, S_mixed(k))（≥k 胜的实际人数是随机变量）
晋级概率 = P(X_k ≤ target_rank - 1)
"安全胜场" = 最小 k 使得晋级概率 ≥ 95%
```
这比 `N × S(k) ≤ target` 更准确，因为它考虑了采样波动。

### 模型验证与局限性

**已有验证**：负二项分布 vs 蒙特卡罗模拟（含配对规则、50% 胜率）的差异在 0-1 胜场以内，在 ChatGPT 对话中通过 100+ 次模拟验证。配对规则（胜场差 ≤ 1）在同质胜率假设下不影响分布。

**模型局限性（用户须知）**：
1. **同质胜率假设**：真实玩家实力有差异，强者在高胜场池更易胜出。这主要影响极头部（前 50 名）排名，对前 900 名门槛影响较小（已通过 Bradley-Terry 模型验证）
2. **退赛行为简化**：真实退赛可能与战绩相关（弱者更易放弃），当前模型假设退赛与战绩无关
3. **截止时间忽略**：假设绝大部分人打到出局。如果截止时间很紧（大量人未打完），模型会失准

**建议使用方式**：将输出视为"基线参考区间"，在边界胜场 ±1 范围内做决策缓冲。

### 禀赋差异模型（Bradley-Terry）

当开启"禀赋差异"模式时，引入玩家实力差异：

```
每人能力 s_i ~ N(0, 1)
对局胜率 P(i胜) = 1 / (1 + e^{-κ × (s_i - s_j)})
κ = 陡峭度参数（默认 1.47，对应顶尖玩家 vs 平均玩家约 95% 胜率）
```

**κ 参考值**：
| 场景 | 顶尖(+2σ) vs 平均(0) 胜率 | κ |
|------|-------------------------|-----|
| 弱差异 | 70% | 0.42 |
| 中等差异 | 85% | 0.95 |
| 强差异（默认） | 95% | 1.47 |
| 极端差异 | 99% | 2.30 |

**对前 900 名门槛的影响**：
- 禀赋差异主要影响**极头部**（冠军、前 50 名）的胜场——强者会集中更多胜场
- 对前 900 名门槛的影响较小但非零，因为强者"吸走"了部分胜场，中段人数分布会略微变化
- **必须通过蒙特卡罗模拟计算**，无简单解析公式

**实现方式**：禀赋模型只在蒙特卡罗模拟中生效。用户可以对比"50% 基线"和"禀赋模拟"两组结果，直观看到差异。

## 功能模块

### 1. 参数面板

| 参数 | 类型 | 默认值 | 范围 |
|------|------|--------|------|
| 参赛人数 N | 滑块 + 输入框 | 240,000 | 1,000 ~ 5,000,000 |
| 命数 r | 下拉选择 | 5 | 3 / 4 / 5 / 6 / 7 |
| 打满比例 α | 滑块 | 0.9 | 0.5 ~ 1.0 |
| 晋级名次 | 输入框 | 900 | 1 ~ N |
| 禀赋差异 κ | 滑块（高级选项） | 1.47 | 0 ~ 3.0（0 = 纯 50% 胜率） |

### 2. 四种查询模式

**模式 1：人数 + 胜场 → 排名**
- 输入：参赛人数、我的胜场（和负场，可选）
- 输出：预估排名、在所有参赛者中的百分位

**模式 2：人数 + 目标名次 → 需要多少胜**
- 输入：参赛人数、目标名次（如 900）
- 输出：每个胜场数对应的晋级概率（如 "14 胜: 72%, 15 胜: 97%, 16 胜: 99.9%"）
- 高亮"95% 安全胜场"（最小 k 使得 P(X_k ≤ target-1) ≥ 95%）
- 算法：X_k ~ Binomial(N, S_mixed(k))，计算 P(X_k ≤ target_rank - 1)

**模式 3：鲁棒性分析（胜场 + 人数范围 → 排名范围）**
- 输入：我的胜场、人数下限、人数上限
- 输出：在不同人数下的排名变化曲线
- 高亮标注"安全区"（排名 ≤ 目标名次）和"危险区"

**模式 4：安全人数上限（胜场 + 目标名次 → 最大安全人数）**
- 输入：我的胜场、目标名次
- 输出：95% 安全人数上限（最大 N 使得晋级概率 ≥ 95%）
- 同时展示 80% 和 99% 安全上限作为参考
- 含标尺展示，标注历史参考点（如"去年 24 万人"）

### 3. 预测最终排名（附 disclaimer）

当输入的负场 < 命数时（如 15-0，还没出局），额外计算：
- 剩余命数 = r - L
- 后续胜场 ~ NegBin(r-L, 0.5)
- 输出最终胜场的概率分布和对应排名区间（50% / 80% / 95% 置信区间）
- 提供可选的"个人胜率假设"滑块（默认 50%，可调到 55%/60%/65%），让用户看到不同实力水平下的预测差异

**UI 上显著标注**："此预测假设后续每局胜率为 50%（总体平均水平）。如果你已经 15-0，你的真实实力很可能高于平均，实际表现会更好。可调整'个人胜率'滑块来做敏感性分析。"

### 4. 可视化图表

**图表 1：胜场分布曲线**
- X 轴：胜场数（0 ~ 最高胜场）
- Y 轴：人数（支持线性/对数刻度切换）
- 柱状图 + 平滑曲线
- 高亮：目标胜场标记、晋级门槛虚线
- Hover tooltip：该胜场恰好多少人、累计 ≥ 该胜场多少人

**图表 2：累计排名曲线**
- X 轴：胜场数
- Y 轴：≥ 该胜场的累计人数（= 排名）
- 晋级线（如前 900）画水平线，交叉点即答案
- 可同时叠加多组参数对比

**图表 3：鲁棒性热力图**
- X 轴：参赛人数范围
- Y 轴：胜场数
- 颜色：排名（绿 = 稳进，黄 = 边缘，红 = 不够）

**图表 4：衰减比例图**
- X 轴：胜场数
- Y 轴：P(W=k+1) / P(W=k) 比值
- 展示"每多赢一场，剩余人数缩减多少比例"

### 5. 数据表

**完整分布表**

| 胜场 | 恰好人数 | ≥该胜场人数 | 排名区间 | 衰减比 |
|------|---------|-----------|---------|-------|
| 0    | ...     | N         | -       | -     |
| 1    | ...     | ...       | ...     | ...   |

**关键节点速查表**
- 前 100 / 500 / 900 / 1000 / 5000 / 10000 名分别需要多少胜

### 6. 数学洞察面板

- 展示核心公式（LaTeX 渲染）
- "每多一胜的衰减规律"解读
- 命数对分布的影响说明

### 7. 蒙特卡罗模拟引擎

当 κ > 0（开启禀赋差异）时，理论公式失效，**必须通过模拟计算**。

- Web Worker 中运行，不阻塞 UI
- 进度条 + 预估剩余时间
- **性能目标**：20 万人单次模拟 ≤ 2 分钟（M5 芯片）
- 实现优化：TypedArray 存储、按胜场排序后线性扫描配对
- 模拟结果与理论基线（κ=0）对比展示

**两种使用场景**：
1. **κ=0 时**：作为理论公式的验证工具（小规模，N ≤ 100,000）
2. **κ>0 时**：作为核心计算手段（支持大规模，N 可达 500,000，模拟 1-10 次取平均）

- **React + TypeScript** — 组件化、类型安全

## 实现阶段

### 阶段 1：数学内核 + 核心查询
- 实现递推公式计算 NegBin 分布
- 实现 Binomial 晋级概率计算
- 实现四种查询模式的计算逻辑
- **单元测试**：归一化验证（所有 P 求和 = 1）、尾分布单调递减、正反查询一致性（模式 2 和模式 4 互为逆运算）
- **退出条件**：所有查询 < 50ms，公式与小规模精确值一致

### 阶段 2：UI + 基础图表
- 参数面板 + 查询卡片
- 分布曲线 + 累计排名曲线
- 完整分布表 + 关键节点速查表
- **退出条件**：核心场景可用，图表正确渲染

### 阶段 3：高级功能
- 鲁棒性热力图
- 衰减比例图
- 数学洞察面板
- 蒙特卡罗模拟验证
- 预测最终排名（含个人胜率滑块）
- **退出条件**：所有功能可用，性能达标

## 不在范围内

- 外部数据校准（需要真实赛事数据，目前无来源）
- 连续赛事串联：每轮赛事独立，用户可手动切换参数
- 后端 / 数据库：纯前端计算，无需服务器
- 用户账号 / 历史记录保存
