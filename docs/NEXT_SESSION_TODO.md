# 下次 Session 改进清单

## 用户明确要求（高优先级）

### 1. Tab 输入重构 — 每个 Tab 自包含所有输入
**当前问题**：参赛人数和目标排名放在 QueryTabs 上方的共享输入栏，用户分不清哪个参数属于哪个 Tab。

**目标设计**：
- **侧边栏只保留**：命数、满局率（真正全局的参数）
- **每个 Tab 内部包含自己需要的所有输入**：
  - 胜场→排名：参赛人数 + 胜场数 → 输出排名
  - 名次→胜场：参赛人数 + 目标名次 → 输出需要的胜场
  - 鲁棒性分析：胜场数 + 人数范围(min/max) + 目标名次 → 输出各人数下的排名
  - 安全人数：胜场数 + 目标名次 → 输出安全人数上限
- **输入和输出在 Tab 内清晰分区**，用户一眼就知道要填什么、结果是什么

### 2. 视觉设计升级 — CR 游戏风格
**用户参考**：皇室战争官方网站 (cr.qq.com) 的设计风格
**关键元素**：
- 可以使用 CR 游戏素材图片（用户确认非商用无版权问题）
- 按钮样式参考游戏内的按钮（圆角、渐变、投影）
- 背景可以使用游戏风格的纹理（菱形网格、石质纹理等 CSS 生成）
- 字体层次更分明：标题粗大醒目，正文清晰舒适
- Pinterest 参考：https://www.pinterest.com/dy0396/游戏美术皇室战争/

### 3. 蒙特卡罗模拟面板增强
- 已支持多次模拟（1/5/10/20/50 次）
- 需要更清晰地展示：
  - 禀赋差异(κ)对结果的影响对比
  - 模拟结果 vs 理论值的差异
  - 让用户直观理解"有实力差异时，前 900 名门槛会怎么变"
